# תיעוד טכני ומדריך פקודות – פרויקט VM Translator


## הסבר על התרגיל:
תרגיל 0 עוסק בבניית תוכנית שמקבלת כקלט תיקייה עם קבצי VM ומתרגמת את הפקודות שבהם לקובץ פלט אחד בשפת Assembly (סיומת .asm). התוכנית קוראת כל קובץ VM בתיקייה, שורה אחר שורה, מפרקת כל שורה למילים ומזהה את סוג הפקודה: אריתמטית, לוגית או פקודת גישה לזיכרון. בהתאם לסוג הפקודה, היא קוראת לפונקציית עזר ייעודית אשר כותבת את הפקודה המתורגמת לקובץ הפלט, ולעיתים מוסיפה מידע נוסף כמו מונה הפקודות הלוגיות. בסיום עיבוד כל קובץ, התוכנית מדפיסה למסך הודעה על סיום קובץ הקלט, ולבסוף, לאחר עיבוד כל קבצי התיקייה, היא מדפיסה הודעה המציינת שהקובץ המלא מוכן. תרגיל זה מדגים עבודה עם קבצים, פירוק שורות, שימוש בפונקציות עזר, ניהול מונים גלובליים, והבנה בסיסית של תרגום משפה ביניים (VM) ל־Assembly.

מסמך זה מפרט את תפקידי הפקודות והלוגיקה ששימשו אותנו לתרגיל שנעשה **Clojure**.
המסמך נועד לספק הסבר מעמיק על אופן פעולת הקוד והתחביר הייחודי שלו.
במסמך זה פירטנו הסבר מה המשמעות של הפקודות אותן רשמנו בקוד.

# 1. ניתוח פונקציות העזר (Helper Functions)

## פקודות לוגיות (כמו <span dir="ltr">handleEq</span> ו־<span dir="ltr">handleGt</span>)

 <span dir="ltr">**defn**</span> – מילת המפתח להגדרת פונקציה חדשה.

 <span dir="ltr">**[writer counter-atom]**</span> – הפרמטרים שהפונקציה מקבלת:
  האובייקט שכותב לקובץ והמונה המשמש ליצירת תוויות ייחודיות.

<span dir="ltr">**swap!**</span> – פקודה המשמשת לעדכון ערך של אטום (**Atom**).
  היא מבטיחה שינוי בטוח של המידע בתוך סביבת הריצה.

<span dir="ltr">**inc**</span> – קיצור של **Increment**, מעלה את הערך המספרי ב־1.

<span dir="ltr">**@**</span> – סימן המשמש לשליפת הערך הנוכחי מתוך האטום (**Dereferencing**).

---

## פקודות זיכרון (כמו <span dir="ltr">handlePus`</span> ו־<span dir="ltr">handlePop</span>)

<span dir="ltr">**str**</span> – פונקציה לשרשור (חיבור) מחרוזות.
  היא הופכת משתנים וטקסט למחרוזת אחת ארוכה.

<span dir="ltr">**write**</span> – קריאה לפונקציית הכתיבה של **Java**
  כדי להכניס את הטקסט לקובץ הפלט המאוחד.

---

# 2. עיבוד טקסט ופירוק שורות (Parsing)

בתוך פונקציית <span dir="ltr">**process-line**</span> אנו מבצעים ניקוי ופירוק של הפקודות:

```clojure
(let [words (str/split (str/trim line) #"\s+")])
```

| פקודה                                  | תפקיד                                                                                  |
| -------------------------------------- | -------------------------------------------------------------------------------------- |
| <span dir="ltr">**str/trim**</span>  | ניקוי רווחים מיותרים, טאבים וירידות שורה מהתחלה ומהסוף של השורה                        |
| <span dir="ltr">**str/split**</span> | חיתוך המחרוזת לרשימה (**List**) של מילים נפרדות                                        |
| <span dir="ltr">**#"\s+"**</span>    | ביטוי רגולרי (**Regex**) המורה למחשב לחתוך את הטקסט בכל פעם שהוא פוגש רווח אחד או יותר |
| <span dir="ltr">**let**</span>       | יצירת סביבה זמנית שבה המשתנה words מחזיק את רשימת המילים שנוצרה                      |

# 3. לוגיקה ותנאים

<span dir="ltr">**when**</span> – גרסה של `if` שמתבצעת רק אם התנאי אמת (ללא צורך ב־`else`).

<span dir="ltr">**and**</span> – אופרטור לוגי שמבצע את הקוד רק אם כל התנאים נכונים.

<span dir="ltr">**seq**</span> – בודקת האם רצף נתונים (כמו רשימת מילים) אינו ריק.

<span dir="ltr">**nil**</span> – ייצוג של ערך ריק או חוסר ערך ב־Clojure.

---

# 4. עבודה עם קבצים (I/O)

<span dir="ltr">**io/file**</span> – הופך מחרוזת טקסט המייצגת נתיב לאובייקט **קובץ** פיזי במערכת.

<span dir="ltr">**with-open**</span> – פתיחת משאב (כמו קורא קבצים) והבטחה שהוא ייסגר אוטומטית
  בסיום העבודה למניעת דליפות זיכרון.

<span dir="ltr">**line-seq**</span> – הופכת קובץ שלם לרצף של שורות שניתן לעבד אחת אחרי השנייה.

<span dir="ltr">**doseq**</span> – לולאה המיועדת לביצוע פעולות עם **תופעות לוואי**
  (כמו כתיבה לקובץ) עבור כל איבר ברשימה.

<span dir="ltr">**.exists / .isDirectory**</span> – פונקציות Java הבודקות האם הנתיב קיים והאם הוא אכן תיקייה.

<span dir="ltr">**.listFiles**</span> – מחזירה רשימה של כל הקבצים הנמצאים בתוך התיקייה שנבחרה.

---

# 5. אינטראקציה עם המשתמש

<span dir="ltr">**& args**</span> – מאפשר לפונקציית `main` לקבל מספר משתנה של פרמטרים משורת הפקודה.

<span dir="ltr">**flush**</span> – ניקוי הזיכרון הזמני (**Buffer**) והבטחה שהטקסט יודפס למסך באופן מיידי.

<span dir="ltr">**read-line**</span> – עוצר את ריצת התוכנית וממתין לקלט מהמשתמש (לחיצה על Enter).

---
# 6.לולאות ורצפים(Iterators)


בפרויקט זה השתמשנו בשני סוגים של לולאות, לכל אחת תפקיד שונה:

doseq (קיצור של Do Sequence)

מה זה עושה? עוברת על איברי רשימה או רצף (כמו רשימת קבצים או שורות בקובץ) ומבצעת פעולה עבור כל איבר.

מתי השתמשנו בה? ב-main כדי לעבור על כל קובצי ה-VM בתיקייה, וב-write-asm כדי לכתוב כל שורת קוד לקובץ.

הערה טכנית: היא מחזירה תמיד nil ומשתמשים בה רק כשרוצים לבצע "תופעות לוואי" (Side Effects) כמו הדפסה למסך או כתיבה לקובץ.

dotimes (קיצור של Do Times)

מה זה עושה? מריצה קוד מספר קבוע של פעמים לפי מספר (Integer) שניתן לה.

מתי השתמשנו בה? בפונקציה handleFunction, כדי לדחוף את הערך 0 למחסנית בדיוק k פעמים (כמספר המשתנים המקומיים שהפונקציה הגדירה).

הערה טכנית: זו הדרך היעילה ביותר ב-Clojure לבצע פעולה שחוזרת על עצמה מספר ידוע של פעמים ללא צורך ברשימה קיימת.

🧠 תיאור הפרויקט

בשלב זה הרחבנו את מתרגם ה־VM כך שיתמוך באופן מלא בפקודות זרימת בקרה (Control Flow) ובניהול פונקציות (Functions).

🔁 זרימת בקרה (Control Flow)

מומשו הפקודות:

label
goto
if-goto

למניעת התנגשויות בין תוויות, כל label נבנה בפורמט:

FileName$Label

פקודת if-goto:

מבצעת pop מהמחסנית
קופצת רק אם הערך שונה מאפס
📞 קריאה לפונקציה (Call)

פקודת call שומרת את מצב הפונקציה הקוראת ומעבירה שליטה לפונקציה החדשה.

שלבי הפעולה:

דחיפת כתובת חזרה (Return Address)
שמירת הסגמנטים:
LCL
ARG
THIS
THAT
חישוב ARG:
ARG = SP - 5 - nArgs
עדכון LCL ל־SP הנוכחי
קפיצה לפונקציה
יצירת תווית חזרה ייחודית
🔙 חזרה מפונקציה (Return)

פקודת return משחזרת את מצב הפונקציה הקוראת:

שלבים:

שמירת FRAME (LCL)
שמירת כתובת החזרה (RET)
העברת ערך החזרה ל־ARG
עדכון:
SP = ARG + 1
שחזור הסגמנטים:
THAT
THIS
ARG
LCL
קפיצה לכתובת החזרה
🧩 הגדרת פונקציה (Function)

פקודת function:

יוצרת תווית התחלה לפונקציה
מאתחלת את המשתנים המקומיים ל־0
דוחפת אותם למחסנית לפי מספר המשתנים
⚡ Bootstrap (קוד אתחול)

בתחילת הריצה המערכת מבצעת אתחול של המחסנית:

@256
D=A
@SP
M=D

לאחר מכן מתבצעת קריאה לפונקציה:

Sys.init

שלב זה הכרחי להפעלת תוכניות הכוללות פונקציות.
