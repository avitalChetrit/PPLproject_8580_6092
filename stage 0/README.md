# תיעוד טכני ומדריך פקודות – פרויקט VM Translator

**מחברים:** טל שזיפי, אביטל חזן

מסמך זה מפרט את תפקידי הפקודות והלוגיקה ששימשו לבניית המתרגם בשפת **Clojure**.
המסמך נועד לספק הסבר מעמיק על אופן פעולת הקוד והתחביר הייחודי שלו.

---

# 1. ניתוח פונקציות העזר (Helper Functions)

## פקודות לוגיות (כמו `handleEq` ו־`handleGt`)

* **`defn`** – מילת המפתח להגדרת פונקציה חדשה.* **`defn`**

* **`[writer counter-atom]`** – הפרמטרים שהפונקציה מקבלת:
  האובייקט שכותב לקובץ והמונה המשמש ליצירת תוויות ייחודיות.

* **`swap!`** – פקודה המשמשת לעדכון ערך של אטום (**Atom**).
  היא מבטיחה שינוי בטוח של המידע בתוך סביבת הריצה.

* **`inc`** – קיצור של **Increment**, מעלה את הערך המספרי ב־1.

* **`@`** – סימן המשמש לשליפת הערך הנוכחי מתוך האטום
  (**Dereferencing**).

---

## פקודות זיכרון (כמו `handlePush` ו־`handlePop`)

* **`str`** – פונקציה לשרשור (חיבור) מחרוזות.
  היא הופכת משתנים וטקסט למחרוזת אחת ארוכה.

* **`write`** – קריאה לפונקציית הכתיבה של **Java**
  כדי להכניס את הטקסט לקובץ הפלט המאוחד.

---

# 2. עיבוד טקסט ופירוק שורות (Parsing)

בתוך פונקציית **`process-line`** אנו מבצעים ניקוי ופירוק של הפקודות:

```clojure
(let [words (str/split (str/trim line) #"\s+")])
```

| פקודה           | תפקיד                                                                                  |
| --------------- | -------------------------------------------------------------------------------------- |
| **`str/trim`**  | ניקוי רווחים מיותרים, טאבים וירידות שורה מהתחלה ומהסוף של השורה                        |
| **`str/split`** | חיתוך המחרוזת לרשימה (**List**) של מילים נפרדות                                        |
| **`#"\s+"`**    | ביטוי רגולרי (**Regex**) המורה למחשב לחתוך את הטקסט בכל פעם שהוא פוגש רווח אחד או יותר |
| **`let`**       | יצירת סביבה זמנית שבה המשתנה `words` מחזיק את רשימת המילים שנוצרה                      |

---

# 3. לוגיקה ותנאים

* **`when`** – גרסה של `if` שמתבצעת רק אם התנאי אמת (ללא צורך ב־`else`).

* **`and`** – אופרטור לוגי שמבצע את הקוד רק אם כל התנאים נכונים.

* **`seq`** – בודקת האם רצף נתונים (כמו רשימת מילים) אינו ריק.

* **`nil`** – ייצוג של ערך ריק או חוסר ערך ב־Clojure.

---

# 4. עבודה עם קבצים (I/O)

* **`io/file`** – הופך מחרוזת טקסט המייצגת נתיב לאובייקט **קובץ** פיזי במערכת.

* **`with-open`** – פתיחת משאב (כמו קורא קבצים) והבטחה שהוא ייסגר אוטומטית
  בסיום העבודה למניעת דליפות זיכרון.

* **`line-seq`** – הופכת קובץ שלם לרצף של שורות שניתן לעבד אחת אחרי השנייה.

* **`doseq`** – לולאה המיועדת לביצוע פעולות עם **תופעות לוואי**
  (כמו כתיבה לקובץ) עבור כל איבר ברשימה.

* **`.exists` / `.isDirectory`** – פונקציות Java הבודקות האם הנתיב קיים והאם הוא אכן תיקייה.

* **`.listFiles`** – מחזירה רשימה של כל הקבצים הנמצאים בתוך התיקייה שנבחרה.

---

# 5. אינטראקציה עם המשתמש

* **`& args`** – מאפשר לפונקציית `main` לקבל מספר משתנה של פרמטרים משורת הפקודה.

* **`flush`** – ניקוי הזיכרון הזמני (**Buffer**) והבטחה שהטקסט יודפס למסך באופן מיידי.

* **`read-line`** – עוצר את ריצת התוכנית וממתין לקלט מהמשתמש (לחיצה על Enter).
